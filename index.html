Visão Geral da Integração SEST SENAT

Seja bem-vindo à documentação base de integração entre o ambiente do SEST SENAT e a plataforma Studion.mx, desenvolvida pela DOT Digital Group.

Este documento serve como guia técnico para orientar a equipe de engenharia no consumo dos serviços, começando pela autenticação da aplicação e a geração de salas virtuais integradas com a infraestrutura do Zoom.

URL Base (Host)

Para realizar as requisições de teste e homologação do projeto, utilize o seguinte endereço de servidor:

https://api-studion-sestsenat.hom.dotgroup.com.br
📝 Nota Importante sobre as Rotas: A URL base descrita acima deve, obrigatoriamente, anteceder (ficar antes) de todos os caminhos relativos dos endpoints listados nesta documentação.

Exemplo prático: Se o endpoint de autenticação está mapeado como /api/login_check, a requisição final contendo o Host completo deverá ser disparada para:
https://api-studion-sestsenat.hom.dotgroup.com.br/api/login_check

Autenticar Usuário

Gera os tokens de acesso (Access Token) e de atualização (Refresh Token) a partir das credenciais do usuário. Este token será necessário para autenticar as requisições nos demais endpoints protegidos da API.

POST /api/login_check

Cabeçalhos (Headers)

Nome Tipo Descrição
Content-Type String Obrigatório. Valor deve ser application/x-www-form-urlencoded.
⚠️ Atenção: Certifique-se de enviar os dados no formato x-www-form-urlencoded. Enviar como application/json resultará em erro.

Corpo da Requisição

Parâmetro Tipo Descrição
_username String Obrigatório. O e-mail/identificador de acesso.
_password String Obrigatório. A senha de acesso.

Respostas

200 OK (Sucesso)

{
    "token": "eyJhbGciOiJSUzI1NiIsInR5...[JWT Omitido]",
    "refresh_token": "eyJhbGciOiJIUzUxMiIsInR5...[Refresh Token Omitido]"
}

Exemplo cURL

curl --location 'https://api-studion-sestsenat.hom.dotgroup.com.br/api/login_check' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode '_username=SEU_USUARIO' \
--data-urlencode '_password=SUA_SENHA'

Criar Conferência (Integração Zoom)

Responsável por criar uma sala de videoconferência integrada entre o Studion e o Zoom. Este endpoint orquestra a geração da sala, horários de acesso e regras de gravação, vinculando diretamente ao host passado no payload.

POST /api/conferences/conferences-rooms

Cabeçalhos (Headers)

Nome Tipo Descrição
Authorization String Obrigatório. Bearer <seu_token_jwt>
Content-Type String Obrigatório. Deve ser enviado como multipart/form-data.
⚠️ Formato de Envio: Diferente de outros endpoints, a criação da sala exige o formato multipart/form-data. Fique atento a isso caso esteja montando o payload programaticamente via Axios, Fetch ou outras bibliotecas HTTP.

Corpo da Requisição (Form Data)

Parâmetro Tipo Descrição
name String Obrigatório. Nome que será exibido na sala (Ex: "Teste Bruno").
start_time Datetime Obrigatório. Data e hora de início no formato YYYY-MM-DD HH:MM.
end_time Datetime Obrigatório. Data e hora de término no formato YYYY-MM-DD HH:MM.
conference_id Integer Obrigatório. ID da estrutura base de conferência.
host String Obrigatório. UUID do usuário anfitrião da sala (Ex: 37ab8cd3-c12d-4d64-8513...).
waiting_room Boolean Opcional. Habilita a sala de espera do Zoom (1 = Sim, 0 = Não).
breakout_room Boolean Opcional. Habilita salas simultâneas/equipes (1 = Sim, 0 = Não).
add_to_calendar Boolean Opcional. Adicionar aula ao vivo no calendário (1 = Sim, 0 = Não).
early_access Boolean Opcional. Permitir acesso antecipado à conferência. Minutos de antecedência permitidos para entrar na sala.
generate_video Boolean Opcional. Permitir acesso a gravação da aula ao vivo (1 = Sim, 0 = Não).
percentage Integer Opcional. Consumo mínimo da gravação Ao finalizar a aula vivo uma aula é gerada automaticamente. Configure a porcentagem para definir a aula ao vivo como concluída.
allow_advance Boolean Opcional. Permitir que o aluno avance o vídeo em conteúdos inéditos (1 = Sim, 0 = Não).
mandatory Boolean Opcional. Defina se a aula será obrigatória para concluir o curso. Se houver gravação e o aluno não assistir ao vivo, a gravação contará como concluída. (1 = Sim, 0 = Não).
image Arquivo Opcional. Imagem da sala da aula ao vivo (Passe arquivo ou `null` / `0` se não utilizado.)
folderId Integer Opcional. ID da pasta onde a aula ao vivo será armazenada. Passe `null` ou `0` se não utilizado.

Respostas

200 OK (Sucesso)

Retorna os detalhes da sala criada, vinculando os identificadores e mostrando a agenda processada no padrão UTC.

500 Internal Server Error

Ocorre quando há problema na geração com o Zoom. O erro mais comum é limite de licenças ativas excedido. 

{
    "code": 500,
    "message": "license_exceeded",
    "hint": []
}

Exemplo cURL

--header 'Accept: application/json' \
--header 'Authorization: Bearer SEU_TOKEN_AQUI' \
--form 'name="Teste Bruno"' \
--form 'image="null"' \
--form 'start_time="2026-05-22 15:00"' \
